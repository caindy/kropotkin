Introduction

Kropotkin is a framework that expressly encourages the practise of Continuous Rewriting. It does this by making it easy (and in fact mandatory) to create your application from a large number of very loosely coupled, miniscule components, which co-operate via a shared factspace containing immutable facts about the business domain.

This page is a "table of contents" for Kropotkin generally and this instance in particular. It links you to pages that describe how Kropotkin works and why works that way, as well as "live" pages showing the current state of this Kropotkin installation.

When you first start Kropotkin, this page is displayed on the root path, typically http://yourdomain.com:2001. Once you deploy one or more applications using Kropotkin, you will normally replace this page with one of your own (by storing a fact of type home_component). Once you do this, you will still be able to find this page at its normal component address, for example http://yourdomain.com:2001/component/kropotkin_home.
