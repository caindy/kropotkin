Kropotkin - Home

Kropotkin is a framework that expressly encourages the practise of Continuous Rewriting. It does this by making it easy (and in fact mandatory) to create your application from a large number of very loosely coupled, miniscule components, which co-operate via a shared factspace containing immutable facts about the business domain.

This page is a both an introduction to Kropotkin and a table of contents of sorts, linking you to various elements of this Kropotkin instance and providing a guide to their usage.

Notice that the page is "live" - as data arrives and processes start or finish, the lists on the page update themselves right away. It's natural to create "live" pages like this in the Kropotkin framework.

Facts

Facts are the basic unit of data in a Kropotkin system. A fact is immutable and public; it should describe something about the domain that your Kropotkin system is designed for.
